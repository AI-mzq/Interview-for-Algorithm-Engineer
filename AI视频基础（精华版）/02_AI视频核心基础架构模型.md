<h1 id="1.文本与多模态条件编码器t5clip及条件注入">1.文本与多模态条件编码器：T5、CLIP 及条件注入</h1>

### 🔑 核心组件
| 编码器 | 类型 | 主要作用 | 特点 |
|--------|------|----------|------|
| T5 | Text Encoder | 文本语义理解 | 语言理解能力强，适合长文本提示词 |
| CLIP | Vision-Language Encoder | 图文语义对齐 | 跨模态对齐能力强，适合视觉语义约束 |

<h2 id="面试问题为什么-ai-视频模型里经常会出现-t5-和-clip">面试问题：为什么 AI 视频模型里经常会出现 T5 和 CLIP？</h2>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐⭐⭐ (5/5)**

AI 视频模型里经常出现 T5 和 CLIP，核心原因是：**视频生成模型本身主要负责生成时空内容，而 T5 和 CLIP 负责把用户输入的文本、图像或多模态条件转换成模型可理解的语义表示。**

从架构上看，视频生成模型通常不是直接读取自然语言 prompt，而是先通过条件编码器得到一组条件特征，再把这些特征注入到扩散模型或 Transformer 主干中。T5 和 CLIP 就是最常见的两类条件编码器。

T5 的优势在于**文本语义理解能力强**。它来自大规模语言建模任务，适合处理较长、较复杂的提示词，能够更好理解句子结构、属性关系、动作描述和组合语义。比如“一个穿红色雨衣的人在夜晚的街道上慢慢转身”这类 prompt，T5 更擅长捕捉其中的修饰关系和动作描述。

CLIP 的优势在于**视觉语义对齐能力强**。它通过图像和文本对比学习训练，文本 embedding 天然和视觉概念空间对齐，因此更适合提供“什么样的画面更符合这段文本”的视觉约束。在图像到视频、参考图驱动视频、风格控制等任务中，CLIP 往往能帮助模型保持更稳定的视觉语义。

所以面试里可以这样回答：**T5 偏语言理解，CLIP 偏视觉对齐。AI 视频模型使用它们，是为了把自然语言或多模态输入转成高质量条件信号，从而提升 prompt following、视觉语义一致性和生成可控性。**

<h2 id="面试问题t5-和-clip-的核心差异是什么">面试问题：T5 和 CLIP 的核心差异是什么？</h2>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

T5 和 CLIP 的核心差异在于：**T5 是语言模型视角的文本编码器，重点是理解文本本身；CLIP 是视觉 - 语言对齐模型，重点是让文本和图像落到可比较的语义空间里。**

可以从几个维度理解：

| 对比维度 | T5 | CLIP |
|----------|----|------|
| 训练目标 | 文本到文本的语言建模或去噪式文本理解 | 图像 - 文本对比学习 |
| 核心能力 | 长文本理解、语义组合、句法关系建模 | 图文语义对齐、视觉概念匹配 |
| 输出特征 | 更偏语言语义表示 | 更偏视觉语义对齐表示 |
| 适合场景 | 复杂 prompt、动作关系、属性组合 | 图文匹配、视觉风格、参考图语义约束 |

T5 更像是“**读懂用户到底说了什么**”。它擅长处理较长的自然语言描述，能保留更细粒度的文本语义，例如主体、动作、属性、场景之间的关系。

CLIP 更像是“**判断这段文字对应什么视觉概念**”。它的文本表示不是单纯为语言理解服务，而是为了和图像表示对齐，因此更适合给生成模型提供视觉语义方向。

需要注意的是，CLIP 并不等价于一个通用大语言模型。它对视觉概念、类别、风格、构图等很敏感，但对复杂逻辑、长文本推理、细粒度动作顺序的理解通常不如专门的语言编码器。

面试中可以总结为：**T5 强在语言理解，CLIP 强在图文对齐；前者帮助模型理解 prompt 的语义结构，后者帮助模型生成更符合视觉语义的内容。**

<h2 id="面试问题为什么有些模型要同时使用-t5-和-clip">面试问题：为什么有些模型要同时使用 T5 和 CLIP？</h2>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

**有些视频生成模型同时使用 T5 和 CLIP**，是因为它们提供的是**互补的条件信息**：T5 补强语言理解，CLIP 补强视觉对齐。对于视频生成这种高维、长时序、多约束任务，只依赖单一条件编码器往往不够稳定。

具体来说，T5 主要解决“prompt 语义是否被读懂”的问题。它能帮助模型理解更长的描述、更复杂的修饰关系和动作过程，例如主体是谁、发生了什么动作、动作顺序如何、场景和风格有什么限制。

CLIP 主要解决“生成画面是否符合视觉语义”的问题。它的表示空间经过图文对齐训练，更容易把文本概念和视觉外观联系起来，例如“cinematic lighting”“oil painting style”“a corgi wearing sunglasses”这类视觉概念。

在实际架构中，两者常见的融合方式包括：

1. **特征拼接或组合**：把 T5 和 CLIP 的文本特征一起作为条件输入。
2. **分别注入不同模块**：例如 T5 hidden states 用于 cross-attention，CLIP pooled embedding 用于全局调制。
3. **多路条件控制**：T5 控制细粒度语义，CLIP 控制视觉风格、主体语义或参考图对齐。

这样做的收益是生成结果通常会有更好的 prompt following 和视觉一致性。尤其在视频里，模型不仅要生成单帧画面，还要让主体、动作和场景在多帧中连续稳定，多源条件可以提供更强的约束。

当然，代价也很明显：同时使用多个编码器会增加显存、推理延迟和工程复杂度，还需要设计好不同条件之间的融合方式，避免语义冲突。

面试里可以回答：**同时使用 T5 和 CLIP，本质上是用 T5 负责“读懂语言”，用 CLIP 负责“对齐视觉”，二者互补可以提升视频生成的语义准确性、视觉一致性和可控性。**

<h2 id="面试问题条件信息通常是如何注入到视频生成模型里的">面试问题：条件信息通常是如何注入到视频生成模型里的？</h2>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐⭐ (5/5)**

条件信息注入指的是：**把文本、图像、音频、动作、姿态、深度图、边缘图等外部控制信号，转化为生成模型内部可使用的特征，并影响去噪或生成过程。**

在 AI 视频生成中，常见的条件注入方式主要有以下几类：

| 注入方式 | 核心思路 | 典型用途 |
|----------|----------|----------|
| Cross-Attention | 让视频 latent 或 token 主动关注文本、多模态条件 token | 文本到视频、图像到视频、多模态生成 |
| 拼接输入 | 将条件图、mask、姿态图等与 noisy latent 在通道或 token 维度拼接 | 图像编辑、局部控制、结构控制 |
| AdaLN / FiLM 调制 | 用条件向量生成归一化层的缩放和平移参数 | DiT 类架构中的全局条件调制 |
| ControlNet / Adapter | 额外分支编码结构条件，再注入主干网络 | 姿态、深度、边缘、轨迹等精细控制 |
| 初始 latent 条件 | 用参考图或首帧编码后的 latent 初始化或约束生成过程 | Image-to-Video、首帧驱动视频生成 |

其中最常见的是 Cross-Attention。文本编码器先把 prompt 编成一组条件 token，视频生成主干中的 latent token 再通过 attention 去读取这些条件，从而知道每个时空位置应该生成什么内容。

对于 DiT 或 Transformer 类视频生成模型，条件也常通过 AdaLN、FiLM 或类似调制方式注入。简单理解，就是条件向量不直接作为 token 参与 attention，而是改变网络中间层的归一化、缩放或偏移参数，从而影响整体生成方向。

对于可控视频生成，条件往往更复杂。例如姿态控制会输入人体骨架，结构控制会输入 depth、canny、segmentation，视频编辑会输入 mask 或参考帧。这类条件通常需要额外编码器或 ControlNet 类分支，避免模型只知道“生成什么”，却不知道“按照什么结构生成”。

面试中可以总结为：**条件注入的本质是把外部控制信号变成生成网络内部的约束。文本语义常用 cross-attention，整体风格或类别可用调制，结构控制常用拼接、Adapter 或 ControlNet。**

<h2 id="面试问题cross-attention-为什么在视频生成里很重要">面试问题：Cross-Attention 为什么在视频生成里很重要？</h2>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

Cross-Attention 在视频生成里很重要，核心原因是：**它提供了一种灵活的条件读取机制，让视频 latent 或时空 token 能够在生成过程中动态关注文本或多模态条件。**

在文本到视频任务中，模型内部处理的是 noisy video latent 或时空 token，而用户输入的是自然语言 prompt。两者模态不同，不能简单相加。Cross-Attention 的作用就是建立二者之间的桥梁：视频 token 作为 query，文本 token 作为 key 和 value，让每个时空位置根据需要读取相关的文本语义。

这带来几个关键好处：

1. **提升 prompt following 能力**：模型可以根据不同 token 关注不同文本片段，例如主体、动作、场景、风格。
2. **支持细粒度语义控制**：相比只用一个全局文本向量，cross-attention 可以保留 token 级语义，更适合复杂 prompt。
3. **适合时空生成任务**：视频中不同帧、不同区域可能对应不同动作和内容，cross-attention 能让每个时空位置动态读取条件。
4. **便于扩展多模态条件**：除了文本，也可以接入图像 token、音频 token、参考帧特征或其他控制信号。

举例来说，如果 prompt 是“a dog running on the beach at sunset”，模型在生成主体区域时需要关注 “dog”，在生成动作时需要关注 “running”，在生成背景和光照时需要关注 “beach” 和 “sunset”。Cross-Attention 允许这些语义在生成过程中被不同位置分别读取，而不是被压缩成一个粗糙的全局条件。

在视频生成中，Cross-Attention 还和时序一致性有关。如果条件注入不稳定，不同帧可能对 prompt 的关注不一致，容易导致主体变化、动作不连贯或风格漂移。因此很多视频模型会结合时空 attention、条件调制和一致性约束来增强跨帧稳定性。

面试里可以回答：**Cross-Attention 是文本或多模态条件和视频生成主干之间的核心连接方式，它让视频 token 能够按需读取条件语义，从而提升 prompt following、细粒度控制和跨帧语义一致性。**

<h1 id="2.vaeldm与潜空间建模">2.VAE、LDM 与潜空间建模</h1>

### 🔑 关键概念
- **VAE（Variational Autoencoder）**：将像素空间视频压缩到更低维的潜空间。
- **LDM（Latent Diffusion Model）**：在潜空间中执行扩散和去噪，降低计算成本。
- **KL Loss**：约束潜变量分布，使潜空间更连续、更可采样。

<h2 id="面试问题什么是-vae它在-ai-视频里有什么作用">面试问题：什么是 VAE？它在 AI 视频里有什么作用？</h2>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐⭐⭐ (5/5)**

VAE（Variational Autoencoder，变分自编码器）是一类**带概率约束的自编码模型**，由编码器、潜变量分布和解码器组成。它的核心思想是：编码器不直接输出一个确定的 latent，而是输出潜变量分布的参数；模型再从这个分布中采样 latent，并由解码器重建原始数据。

对于图像或视频样本 $x$，VAE 通常建模为：

$$
q_\phi(z \mid x) = \mathcal{N}(\mu_\phi(x), \sigma_\phi^2(x) I)
$$

其中 $q_\phi(z \mid x)$ 是编码器给出的近似后验分布， $\mu_\phi(x)$ 和 $\sigma_\phi(x)$ 分别表示均值和标准差。采样时通常使用重参数化技巧：

$$
z = \mu_\phi(x) + \sigma_\phi(x) \odot \epsilon, \qquad \epsilon \sim \mathcal{N}(0, I)
$$

这样可以在保留随机采样的同时，让梯度能够从解码器反传到编码器。

VAE 的训练目标通常来自 ELBO（Evidence Lower Bound），可以写成：

$$
\mathcal{L}_{VAE}
= \mathbb{E}_{q_\phi(z \mid x)}[-\log p_\theta(x \mid z)] + \beta D_{KL}(q_\phi(z \mid x) \Vert p(z))
$$

其中第一项是重建损失，要求解码器能从 latent 还原原始图像或视频；第二项是 KL 正则项，要求编码后的潜变量分布接近先验分布 $p(z)$，通常取标准高斯分布 $\mathcal{N}(0, I)$。

在 AI 视频生成里，VAE 的作用非常关键：**它负责把高维像素空间的视频压缩到低维潜空间，并在生成结束后把潜变量解码回视频帧。**

典型流程可以概括为：

```text
原始视频 x -> VAE Encoder -> 视频 latent z
生成模型在 z 空间中建模或去噪
生成后的 latent -> VAE Decoder -> 输出视频
```

在 Stable Diffusion、Latent Diffusion Models、Stable Video Diffusion、AnimateDiff、VideoCrafter 等模型体系中，VAE 或类似 autoencoder 都承担了“像素空间和潜空间之间的桥梁”角色。

面试里可以回答：**VAE 是一种概率自编码器，在 AI 视频里主要用于压缩和重建视频。它把像素级视频映射到更紧凑的 latent space，让后续扩散模型不必直接在巨大像素空间中建模，从而显著降低计算成本。**

<h2 id="面试问题为什么视频扩散模型通常要在潜空间建模">面试问题：为什么视频扩散模型通常要在潜空间建模？</h2>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐⭐⭐ (5/5)**

视频扩散模型通常在潜空间建模，核心原因是：**直接在像素空间对视频做扩散代价太高，而潜空间可以大幅降低时空维度上的计算量，同时保留主要语义和结构信息。**

视频数据的维度非常高。一个视频样本通常可以表示为：

$$
x \in \mathbb{R}^{T \times C \times H \times W}
$$

其中 $T$ 是帧数， $C$ 是通道数， $H$ 和 $W$ 是空间分辨率。如果直接在像素空间做扩散，噪声预测网络需要同时处理大量时空像素，显存、算力和训练成本都会非常高。

LDM（Latent Diffusion Model）的核心思想是：**先用 autoencoder 或 VAE 把数据压缩到 latent space，再在 latent space 中做扩散建模。**

可以写成：

$$
z = E(x), \qquad \hat{x} = D(z)
$$

扩散过程不再作用于原始像素 $x$，而是作用于潜变量 $z$：

$$
z_t = \sqrt{\bar{\alpha}_t} z_0 + \sqrt{1 - \bar{\alpha}_t} \epsilon, \qquad \epsilon \sim \mathcal{N}(0, I)
$$

训练时，噪声预测网络学习从带噪 latent 中预测噪声：

$$
\mathcal{L}_{LDM}
= \mathbb{E}_{z_0, t, \epsilon}
\left[
\left\| \epsilon - \epsilon_\theta(z_t, t, c) \right\|_2^2
\right]
$$

其中 $c$ 表示文本、图像或其他条件信息， $\epsilon_\theta$ 是扩散模型的噪声预测网络。

潜空间建模的优势主要有三点：

1. **显著降低计算成本**：空间分辨率通常被压缩到原来的 $1/4$、 $1/8$ 甚至更低，视频任务中节省尤其明显。
2. **保留更高层语义**：latent 通常包含主体、结构、纹理和运动相关信息，比纯像素更适合生成模型学习。
3. **提升工程可扩展性**：高分辨率、长视频、多条件控制都更容易在 latent space 中实现。

典型模型包括 Latent Diffusion Models（LDM）、Stable Diffusion、Stable Video Diffusion、VideoCrafter、ModelScope Text-to-Video 等。它们的共同思想都是：**将生成难题从像素空间转移到更紧凑的潜空间。**

面试中可以总结为：**视频扩散模型使用潜空间建模，是为了在保证语义表达能力的同时降低时空计算量。VAE 负责压缩和解码，扩散模型负责在 latent space 中学习生成分布，这是现代高效视频生成架构的基础。**

<h2 id="面试问题vae-的-kl-loss-是干什么的">面试问题：VAE 的 KL Loss 是干什么的？</h2>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐ (3/5)**

VAE 的 KL Loss 用来约束编码器输出的潜变量分布，使其接近预设先验分布，通常是标准高斯分布。它的核心作用是：**让 latent space 连续、规整、可采样，而不是变成一堆离散且无结构的编码点。**

在 VAE 中，编码器输出的是近似后验分布：

$$
q_\phi(z \mid x) = \mathcal{N}(\mu, \sigma^2 I)
$$

先验分布通常设为：

$$
p(z) = \mathcal{N}(0, I)
$$

KL Loss 衡量二者之间的分布差异：

$$
D_{KL}(q_\phi(z \mid x) \Vert p(z))
$$

对于对角高斯分布，KL 项可以写成常见的闭式形式：

$$
D_{KL}
= \frac{1}{2} \sum_i
\left(
\mu_i^2 + \sigma_i^2 - \log \sigma_i^2 - 1
\right)
$$

如果没有 KL Loss，编码器可能会把不同样本映射到非常分散、不可控的 latent 区域，解码器虽然能重建训练样本，但 latent space 不连续、不平滑，生成模型很难在其中学习稳定分布。

KL Loss 的作用可以从三个角度理解：

1. **约束潜空间分布**：让编码结果不要偏离标准高斯太远。
2. **增强可采样性**：从先验分布采样 latent 时，解码器更可能生成合理样本。
3. **提升连续性和平滑性**：相近 latent 对应相近内容，便于扩散模型在潜空间中去噪。

但 KL Loss 也不能过强。如果 KL 权重太大，模型会过度压缩信息，导致重建模糊、细节丢失，甚至出现 posterior collapse。实际工程里通常需要在重建质量和潜空间规整性之间做权衡。

面试里可以回答：**KL Loss 的本质是让 VAE 的编码分布接近标准高斯先验，使 latent space 更连续、更平滑、更适合采样和扩散建模；但 KL 约束过强会损伤重建质量。**

<h2 id="面试问题为什么说-vae-会带来信息损失">面试问题：为什么说 VAE 会带来信息损失？</h2>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

说 VAE 会带来信息损失，主要是因为它不是简单地无损压缩，而是把高维像素数据压缩到更低维、更规整的潜空间中。这个过程必然会丢掉一部分难以压缩或被模型认为不重要的细节。

从编码 - 解码过程看：

$$
z = E(x), \qquad \hat{x} = D(z)
$$

理想情况下希望 $\hat{x} \approx x$，但由于 latent 维度更低、编码器容量有限、KL 正则存在，重建结果通常不可能完全等于原始输入。

VAE 的信息损失主要来自几个方面：

1. **空间压缩损失**：VAE 通常会下采样图像或视频，把高分辨率像素压缩成低分辨率 latent，细纹理、文字、小物体容易丢失。
2. **通道瓶颈损失**：latent 的通道数和表达容量有限，无法完整保留所有像素级信息。
3. **KL 正则损失**：为了让 latent 分布接近标准高斯，模型会牺牲一部分重建精度来换取潜空间规整性。
4. **解码器重建误差**：解码器本身可能无法完美恢复高频纹理、锐利边缘和复杂运动细节。

在图像生成中，VAE 信息损失常表现为：小字不清晰、纹理发糊、边缘不够锐利。
在视频生成中，还会进一步表现为：细节闪烁、局部纹理不稳定、快速运动区域模糊、跨帧细节不一致。

这也是为什么现代生成模型中 VAE 的设计非常关键。比如 Stable Diffusion 系列依赖高质量图像 VAE，Stable Video Diffusion、VideoCrafter 等视频模型则需要更关注时间维度上的重建一致性。如果 VAE 本身重建能力不足，后面的扩散主干再强，也很难恢复已经丢失的细节。

面试中可以总结为：**VAE 的信息损失来自压缩瓶颈、KL 正则和解码误差。它用一定的细节损失换取低维、连续、可建模的 latent space；在视频任务中，这种损失不仅影响单帧清晰度，还会影响跨帧一致性。**

<h2 id="面试问题视频-vae-和图像-vae-有什么不同">面试问题：视频 VAE 和图像 VAE 有什么不同？</h2>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

视频 VAE 和图像 VAE 的核心不同在于：**图像 VAE 只需要压缩和重建单帧空间内容，而视频 VAE 还必须处理时间维度，保证多帧重建的一致性和运动连续性。**

图像 VAE 的输入通常是：

$$
x \in \mathbb{R}^{C \times H \times W}
$$

视频 VAE 的输入通常是：

$$
x \in \mathbb{R}^{T \times C \times H \times W}
$$

因此视频 VAE 不仅要压缩空间维度 $H, W$，还要决定如何处理时间维度 $T$。不同模型会有不同设计：

| 类型 | 核心思路 | 特点 |
|------|----------|------|
| 逐帧图像 VAE | 对每一帧独立编码和解码 | 简单、可复用图像 VAE，但时序一致性弱 |
| 2D + Temporal 模块 | 空间用 2D CNN，时间维度加时序卷积或 attention | 在效率和时序建模之间折中 |
| 3D VAE | 使用 3D 卷积同时压缩时间和空间 | 更适合视频重建，但计算成本更高 |
| Causal Video VAE | 只依赖当前帧及历史帧进行编码或解码 | 适合流式生成和长视频建模 |

视频 VAE 的关键挑战包括：

1. **时序一致性**：相邻帧重建后不能出现颜色、纹理、边缘的随机抖动。
2. **运动保真**：快速运动、遮挡、形变区域不能被过度平滑。
3. **压缩率选择**：空间压缩可以降低成本，但时间压缩如果过强，容易丢失动作细节。
4. **长视频扩展**：视频越长，VAE 的显存和时序稳定性压力越大。

典型模型中，早期很多视频生成方法直接复用图像 VAE，对每帧独立编码，这种方式实现简单，但容易出现 flicker。更强的视频生成系统会使用视频专用 autoencoder，例如包含 3D 卷积、temporal attention 或 causal temporal compression 的 Video VAE，以更好保留跨帧结构。

如果从 latent 形状看，图像 latent 可能是：

$$
z \in \mathbb{R}^{C_z \times H' \times W'}
$$

而视频 latent 通常是：

$$
z \in \mathbb{R}^{T' \times C_z \times H' \times W'}
$$

其中 $T'$ 可能等于 $T$，也可能经过时间下采样。时间维度是否压缩，是视频 VAE 设计中的重要选择。

面试里可以总结为：**图像 VAE 解决的是空间压缩和重建，视频 VAE 还要解决时间维度上的压缩、运动保真和跨帧一致性。视频 VAE 设计得不好，会直接导致生成视频模糊、闪烁和时序不稳定。**

<h1 id="3.video-diffusion-models噪声建模与采样流程">3.Video Diffusion Models：噪声建模与采样流程</h1>

### 🔑 扩散过程
```text
前向过程（加噪）：x0 -> x1 -> ... -> xt
反向过程（去噪）：xt -> xt-1 -> ... -> x0
```

<h2 id="面试问题什么是-video-diffusion-models">面试问题：什么是 Video Diffusion Models？</h2>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐⭐⭐ (5/5)**

Video Diffusion Models 指的是：**把扩散模型应用到视频生成任务中，通过逐步加噪和逐步去噪来学习视频数据分布的生成模型。**

它的基本思想和图像扩散模型一致：训练时把干净视频逐步加噪，直到接近高斯噪声；生成时从随机噪声出发，通过神经网络逐步去噪，最终得到一段符合条件的视频。

如果视频在像素空间中表示为：

$$
x_0 \in \mathbb{R}^{T \times C \times H \times W}
$$

在潜空间视频扩散中，更常见的是先用 VAE 编码成：

$$
z_0 = E(x_0), \qquad z_0 \in \mathbb{R}^{T' \times C_z \times H' \times W'}
$$

然后扩散模型在 $z_0$ 上执行加噪和去噪。现代视频生成模型通常采用这种 latent video diffusion 形式，而不是直接在原始像素空间中扩散。

Video Diffusion Models 的核心组成通常包括：

| 组件 | 作用 |
|------|------|
| VAE / Video VAE | 将视频压缩到 latent space，并在最后解码回视频 |
| 噪声调度器 | 定义每一步加噪和去噪的强度 |
| 噪声预测网络 | 预测当前 noisy latent 中的噪声或其他目标 |
| 条件编码器 | 编码文本、图像、音频、首帧、动作等条件 |
| 采样器 | 按 DDPM、DDIM、DPM-Solver 等策略执行反向去噪 |

与图像扩散相比，视频扩散最大的不同在于输入多了时间维度。模型不仅要生成单帧画面，还要保证帧间运动连续、主体身份稳定、场景结构一致。因此视频扩散模型通常需要引入 3D U-Net、Temporal Attention、Space-Time Attention、DiT / Video Transformer、3D RoPE 等时空建模机制。

典型模型包括 Video Diffusion Models、Imagen Video、Make-A-Video、ModelScope Text-to-Video、VideoCrafter、Stable Video Diffusion、AnimateDiff、Sora 类视频生成模型等。它们的共同点是：**用扩散过程建模视频分布，用时空网络处理视频的时间和空间结构。**

面试里可以回答：**Video Diffusion Models 是用扩散生成框架来生成视频的模型。它通过前向加噪学习视频数据的噪声扰动过程，再通过反向去噪从随机噪声恢复视频；相比图像扩散，关键难点在于时序建模、运动一致性和长程依赖。**

<h2 id="面试问题视频扩散模型的前向加噪过程是如何工作的">面试问题：视频扩散模型的前向加噪过程是如何工作的？</h2>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

视频扩散模型的前向加噪过程，本质上是一个固定的马尔可夫过程：**从干净视频 latent 开始，在每个时间步逐渐加入高斯噪声，最后把数据变成接近标准高斯噪声的形式。**

假设干净视频 latent 为 $z_0$，前向扩散过程可以写成：

$$
q(z_t \mid z_{t-1})
= \mathcal{N}(\sqrt{1-\beta_t} z_{t-1}, \beta_t I)
$$

其中 $\beta_t$ 是第 $t$ 步的噪声强度，通常由 noise schedule 预先设定。定义：

$$
\alpha_t = 1 - \beta_t, \qquad \bar{\alpha}_t = \prod_{s=1}^{t} \alpha_s
$$

则可以直接从 $z_0$ 采样任意时间步的带噪 latent：

$$
q(z_t \mid z_0)
= \mathcal{N}(\sqrt{\bar{\alpha}_t} z_0, (1-\bar{\alpha}_t) I)
$$

等价地，可以写成常见的闭式加噪公式：

$$
z_t = \sqrt{\bar{\alpha}_t} z_0 + \sqrt{1-\bar{\alpha}_t} \epsilon,
\qquad \epsilon \sim \mathcal{N}(0, I)
$$

这个公式非常重要，面试中经常会问。它说明训练时不需要一步步真的从 $z_0$ 加噪到 $z_t$，而是可以随机采样一个时间步 $t$，直接构造对应强度的 noisy latent。

在视频扩散中， $z_t$ 仍然保留视频的时空结构，例如：

$$
z_t \in \mathbb{R}^{T' \times C_z \times H' \times W'}
$$

噪声通常会加到整个视频 latent 上，也就是每一帧、每个空间位置、每个通道都会被扰动。随着 $t$ 变大，视频中的结构、纹理和运动信息逐渐被噪声淹没；当 $t$ 接近最大时间步时， $z_t$ 基本接近高斯噪声。

前向过程本身不需要学习，它是人为设计好的固定过程。模型真正要学习的是反向过程，即如何从 $z_t$ 一步步恢复到 $z_0$。

面试里可以总结为：**前向加噪就是按照预设噪声调度，把干净视频 latent 逐步扰动成高斯噪声。** 关键公式是 $z_t = \sqrt{\bar{\alpha}_t} z_0 + \sqrt{1-\bar{\alpha}_t}\epsilon$，它允许训练时直接构造任意噪声等级的样本。

<h2 id="面试问题视频扩散模型的反向去噪过程是如何实现的">面试问题：视频扩散模型的反向去噪过程是如何实现的？</h2>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

视频扩散模型的反向去噪过程，是指：**从一个随机高斯噪声视频 latent 出发，通过神经网络逐步预测并去除噪声，最终恢复出清晰的视频 latent，再由 VAE 解码成视频。**

理论上，反向过程可以写成：

$$
p_\theta(z_{t-1} \mid z_t, c)
= \mathcal{N}(\mu_\theta(z_t, t, c), \Sigma_\theta(z_t, t, c))
$$

其中 $c$ 是条件信息，例如文本 prompt、参考图、首帧、姿态或音频； $\mu_\theta$ 和 $\Sigma_\theta$ 是模型预测的反向分布参数。

实际工程中，最常见的做法不是直接预测 $z_{t-1}$，而是训练网络预测噪声 $\epsilon$：

$$
\epsilon_\theta(z_t, t, c) \approx \epsilon
$$

有了预测噪声后，可以先估计干净 latent：

$$
\hat{z}_0
= \frac{z_t - \sqrt{1-\bar{\alpha}_t}\epsilon_\theta(z_t, t, c)}
{\sqrt{\bar{\alpha}_t}}
$$

再根据采样器规则从 $z_t$ 更新到 $z_{t-1}$。DDPM 的反向均值常写成：

$$
\mu_\theta(z_t, t, c)
= \frac{1}{\sqrt{\alpha_t}}
\left(
z_t - \frac{\beta_t}{\sqrt{1-\bar{\alpha}_t}}
\epsilon_\theta(z_t, t, c)
\right)
$$

然后进行采样：

$$
z_{t-1} = \mu_\theta(z_t, t, c) + \sigma_t \eta,
\qquad \eta \sim \mathcal{N}(0, I)
$$

其中 $\sigma_t$ 控制采样随机性。如果使用 DDIM 或一些 ODE/Solver 类采样器，反向过程可以更接近确定性，并用更少步数完成生成。

视频扩散的反向去噪比图像更难，因为每一步去噪都必须同时满足三类要求：

1. **单帧质量**：每一帧要清晰、自然、结构合理。
2. **时序一致性**：相邻帧之间不能闪烁、跳变或身份漂移。
3. **条件对齐**：整体视频要符合文本、首帧、动作或其他控制条件。

因此，反向去噪网络通常需要时空联合建模能力，例如 3D U-Net、Temporal Attention、Space-Time Transformer 或 DiT 类架构。仅靠逐帧去噪，很容易得到“单帧还行，但播放起来不连贯”的视频。

面试中可以回答：**反向去噪就是从随机视频噪声 latent 开始，利用噪声预测网络逐步估计并去除噪声，最终得到干净视频 latent。** 核心是学习 $p_\theta(z_{t-1} \mid z_t, c)$，工程上通常通过预测噪声 $\epsilon_\theta(z_t,t,c)$ 来实现。

<h2 id="面试问题视频扩散模型中的噪声预测网络如何设计">面试问题：视频扩散模型中的噪声预测网络如何设计？</h2>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

视频扩散模型中的噪声预测网络，是整个模型的生成主干。它的任务是：**输入带噪视频 latent、扩散时间步和条件信息，输出当前噪声、干净 latent 或 velocity 等预测目标。**

最常见的形式是噪声预测：

$$
\epsilon_\theta(z_t, t, c)
$$

其中 $z_t$ 是带噪视频 latent， $t$ 是扩散时间步， $c$ 是条件信息。网络必须根据当前噪声等级和条件信息，判断哪些部分是噪声、哪些部分应该被保留或恢复。

视频噪声预测网络通常要解决两个维度的问题：

| 维度 | 需要建模的内容 |
|------|----------------|
| 空间维度 | 单帧中的物体、纹理、背景、构图和细节 |
| 时间维度 | 帧间运动、身份一致性、动作连续性和长程依赖 |

常见设计路线包括：

| 架构类型 | 核心思想 | 代表特点 |
|----------|----------|----------|
| 3D U-Net | 用 3D 卷积同时处理时间和空间 | 局部时空建模强，结构接近图像扩散 U-Net |
| 2D U-Net + Temporal Module | 复用图像 U-Net，在中间层插入时间卷积或 temporal attention | 便于从图像模型迁移到视频模型 |
| Video Transformer / DiT | 把视频 latent 切成时空 token，用 Transformer 建模 | 全局依赖建模强，适合大规模视频生成 |
| Factorized Attention | 分别做空间 attention 和时间 attention | 降低计算成本，常用于长视频或高分辨率视频 |

噪声预测网络通常还需要注入三类信息：

1. **时间步信息**：通过 timestep embedding 告诉模型当前噪声强度。
2. **条件信息**：通过 cross-attention、AdaLN、FiLM、拼接或 ControlNet 等方式注入文本、图像、姿态等条件。
3. **位置和时序信息**：通过 2D/3D positional embedding、RoPE、3D RoPE 等方式编码帧位置和空间位置。

在视频任务中，一个关键设计原则是：**不能只把每帧当作独立图像去噪，而要让网络显式或隐式地看到多帧之间的关系。** 否则生成结果容易出现 flicker、身份漂移、动作不连贯等问题。

典型模型中，早期视频扩散常使用 3D U-Net 或在图像 U-Net 中插入 temporal attention；AnimateDiff 的核心思路就是给已有图像扩散模型添加 motion module；Stable Video Diffusion、VideoCrafter 等模型也会围绕时空注意力、视频 latent 和条件注入设计噪声预测主干；更大规模模型则逐渐转向 DiT / Transformer 风格。

面试里可以总结为：**视频扩散的噪声预测网络不仅要预测噪声，还要同时建模空间细节和时间动态。主流设计包括 3D U-Net、2D U-Net 加时序模块、Video Transformer / DiT，以及分解式时空注意力。**

<h2 id="面试问题视频扩散模型的训练目标通常是什么">面试问题：视频扩散模型的训练目标通常是什么？</h2>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

视频扩散模型的训练目标通常是：**随机选择一个扩散时间步，把干净视频 latent 加噪到该时间步，然后训练网络预测被加入的噪声或等价目标。**

最经典的是 DDPM 的噪声预测目标：

$$
\mathcal{L}_{simple}
= \mathbb{E}_{z_0, t, \epsilon, c}
\left[
\left\| \epsilon - \epsilon_\theta(z_t, t, c) \right\|_2^2
\right]
$$

其中：

- $z_0$ 是干净视频 latent
- $t$ 是随机采样的扩散时间步
- $\epsilon \sim \mathcal{N}(0, I)$ 是真实加入的高斯噪声
- $c$ 是条件信息
- $\epsilon_\theta$ 是噪声预测网络

训练流程可以概括为：

```text
采样视频 x0 -> VAE 编码得到 z0
随机采样时间步 t 和噪声 epsilon
构造 zt = sqrt(alpha_bar_t) z0 + sqrt(1-alpha_bar_t) epsilon
输入 zt、t、条件 c
训练网络预测 epsilon
```

除了预测噪声 $\epsilon$，现代扩散模型也常见其他预测参数化方式：

| 预测目标 | 含义 | 特点 |
|----------|------|------|
| $\epsilon$ prediction | 预测加入的噪声 | DDPM 中最经典，稳定易用 |
| $x_0$ / $z_0$ prediction | 直接预测干净样本或干净 latent | 直观，但不同噪声等级下尺度处理更敏感 |
| $v$ prediction | 预测 velocity 形式的目标 | 在一些模型中训练和采样更稳定 |

常见的 $v$ prediction 可以写成：

$$
v = \sqrt{\bar{\alpha}_t}\epsilon - \sqrt{1-\bar{\alpha}_t} z_0
$$

对应训练目标为：

$$
\mathcal{L}_{v}
= \mathbb{E}
\left[
\left\| v - v_\theta(z_t, t, c) \right\|_2^2
\right]
$$

视频扩散模型除了基本去噪损失，有时还会加入额外约束，例如：

1. **时序一致性损失**：约束相邻帧或特征在时间上稳定。
2. **重建或感知损失**：提升视觉质量和语义保真。
3. **运动约束**：利用光流、轨迹或姿态增强运动合理性。
4. **条件对齐损失**：提升文本、图像或动作条件的跟随能力。

但在大多数基础扩散模型中，核心训练目标仍然是去噪预测损失。其他损失更多是针对特定任务或工程需求的增强项。

面试中可以回答：**视频扩散模型最核心的训练目标是 denoising score matching，工程上通常表现为预测加入到视频 latent 中的噪声。** 具体来说，就是最小化 $\|\epsilon - \epsilon_\theta(z_t,t,c)\|_2^2$。也可以采用 $z_0$ prediction 或 $v$ prediction，并根据任务加入时序一致性或条件对齐约束。

<h2 id="面试问题classifier-free-guidance-在视频生成中起什么作用">面试问题：Classifier-Free Guidance 在视频生成中起什么作用？</h2>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐ (3/5)**

Classifier-Free Guidance（CFG）是一种在不额外训练分类器的情况下增强条件控制的方法。它的核心作用是：**在采样阶段放大条件信息的影响，让生成视频更符合文本、图像或其他控制条件。**

CFG 的训练方式很简单：训练时以一定概率丢弃条件，让同一个模型同时学会条件生成和无条件生成。也就是说，模型既能预测：

$$
\epsilon_\theta(z_t, t, c)
$$

也能预测：

$$
\epsilon_\theta(z_t, t, \varnothing)
$$

其中 $\varnothing$ 表示空条件或无条件输入。

采样时，将有条件预测和无条件预测组合起来：

$$
\hat{\epsilon}_\theta
= \epsilon_\theta(z_t, t, \varnothing) + s \left(
\epsilon_\theta(z_t, t, c) - \epsilon_\theta(z_t, t, \varnothing)
\right)
$$

其中 $s$ 是 guidance scale。也可以写成等价形式：

$$
\hat{\epsilon}_\theta
= (1+s)\epsilon_\theta(z_t, t, c) - s\epsilon_\theta(z_t, t, \varnothing)
$$

CFG 的直观理解是：无条件预测表示“模型自然会生成什么”，条件预测表示“在 prompt 约束下应该生成什么”。两者之差可以看作条件方向，guidance scale 就是在采样时放大这个方向。

在视频生成中，CFG 主要有几个作用：

1. **增强 prompt following**：让生成结果更贴近文本描述或多模态条件。
2. **提高主体和风格一致性**：减少模型偏离条件生成无关内容的概率。
3. **增强可控性**：用户可以通过调节 guidance scale 控制条件强度。

但 CFG 不是越大越好。guidance scale 过小，视频可能不听 prompt；guidance scale 过大，可能导致过饱和、细节伪影、运动僵硬、画面不自然，甚至破坏时序一致性。视频任务中这个问题更明显，因为过强条件约束可能让每帧都强行贴合 prompt，却牺牲自然运动和跨帧连续性。

典型实践中，文本到图像和文本到视频模型都会使用 CFG。Stable Diffusion、Imagen、VideoCrafter、Stable Video Diffusion 等扩散体系中，CFG 都是提升条件生成质量的重要采样技巧。

面试里可以总结为：**Classifier-Free Guidance 是采样阶段增强条件控制的技术。它通过组合有条件和无条件噪声预测，放大条件方向，从而提升 prompt following 和可控性；但 guidance scale 过大可能带来伪影、过拟合条件和视频时序不稳定。**

<h1 id="4.ditfitu-net扩散主干网络怎么选">4.DiT、FiT、U-Net：扩散主干网络怎么选</h1>

### 🔑 主干网络对比
| 架构 | 全称 | 核心特点 | 常见场景 |
|------|------|----------|----------|
| U-Net | U-Net | 多尺度卷积、跳跃连接 | 早期图像/视频扩散模型 |
| DiT | Diffusion Transformer | Token 化建模、全局注意力 | 大规模高质量生成 |
| FiT | Full-image Transformer | 更强调全图/全帧 Transformer 表示 | 高分辨率生成与统一建模 |

<h2 id="面试问题u-net-为什么会成为早期扩散模型的主干">面试问题：U-Net 为什么会成为早期扩散模型的主干？</h2>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

U-Net 会成为早期扩散模型的主干，核心原因是：**扩散模型本质上是一个逐步去噪任务，而 U-Net 的多尺度编码 - 解码结构非常适合做图像或视频 latent 的噪声预测。**

U-Net 最初用于医学图像分割，其结构特点是：

```text
输入 -> 下采样编码器 -> bottleneck -> 上采样解码器 -> 输出
             |                         ^
             |------ skip connection ---|
```

对于扩散模型来说，输入是带噪图像或带噪 latent，输出通常是噪声预测：

$$
\epsilon_\theta(x_t, t, c)
$$

这里 $x_t$ 或 $z_t$ 包含不同尺度的噪声和结构信息。U-Net 的优势正好对应这个任务需求：

1. **多尺度建模能力强**：下采样路径捕捉全局结构，上采样路径恢复局部细节。
2. **跳跃连接保留细节**：skip connection 能把浅层高分辨率特征直接传到解码端，适合恢复边缘、纹理和空间结构。
3. **卷积归纳偏置强**：卷积天然适合处理局部空间相关性，对图像生成任务非常有效。
4. **计算效率较高**：相比早期全局 Transformer，卷积 U-Net 在中低分辨率 latent 上更容易训练和部署。
5. **便于条件注入**：可以在不同层加入 timestep embedding、cross-attention、ControlNet、Adapter 等条件模块。

在 Stable Diffusion 这类 Latent Diffusion Model 中，U-Net 是噪声预测网络的核心。输入 noisy latent、扩散时间步和文本条件，输出预测噪声。视频扩散的早期方法也常在图像 U-Net 的基础上加入 temporal convolution、temporal attention 或 motion module，把 2D U-Net 扩展到视频场景。

不过，**U-Net 的局限也很明显**：它更依赖局部卷积和多尺度特征融合，对于长程依赖、全局 token 交互和大规模视频时空建模不如 Transformer 灵活。

因此在更大规模、更长视频、更高分辨率的模型中，主干逐渐向 DiT / Video Transformer 演进。

面试里可以回答：**U-Net 成为早期扩散主干，是因为它的多尺度编码 - 解码结构和扩散去噪任务高度匹配，既能保留局部细节，又能建模全局结构；但在大规模视频生成中，U-Net 的长程时空建模能力逐渐成为瓶颈。**

<h2 id="面试问题dit-是什么为什么视频生成逐渐转向-transformer">面试问题：DiT 是什么？为什么视频生成逐渐转向 Transformer？</h2>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐⭐ (5/5)**

DiT（Diffusion Transformer）指的是：**用 Transformer 作为扩散模型噪声预测主干的架构范式。** 它把图像或视频 latent 切成 token，然后通过 Transformer block 进行全局建模，最后预测噪声、干净样本或 velocity。

以 latent diffusion 为例，假设视频 latent 为：

$$
z_t \in \mathbb{R}^{T' \times C_z \times H' \times W'}
$$

DiT / Video DiT 通常会把它切成时空 patch token：

$$
z_t \rightarrow \{p_1, p_2, \dots, p_N\}
$$

其中每个 token 表示一个空间 patch 或时空 patch。之后通过 Transformer 进行建模：

$$
h^{l+1} = \mathrm{TransformerBlock}(h^l, t, c)
$$

其中 $t$ 是扩散时间步， $c$ 是文本、图像或其他条件信息。

视频生成逐渐转向 Transformer，主要有几个原因：

1. **全局依赖建模更强**：Self-Attention 可以让远距离 token 直接交互，更适合长视频、复杂动作和全局一致性。
2. **更适合大规模训练**：Transformer 在语言模型和视觉模型中已经证明了很好的 scaling behavior，参数量、数据量和算力增加时性能通常更可预测。
3. **统一多模态建模更自然**：文本 token、图像 token、视频 token、音频 token 都可以在 token space 中统一处理。
4. **适合时空联合建模**：视频可以被表示成 space-time tokens，Transformer 可以同时建模空间结构和时间动态。
5. **架构更简洁统一**：相比卷积 U-Net 中复杂的多尺度设计，DiT 更接近通用 token 建模框架。

典型图像生成模型中，DiT 代表了从 U-Net 向 Transformer 扩散主干的转变；在视频生成中，Sora 类模型、许多 Video DiT / Diffusion Transformer 系统也体现了类似趋势：把视频压缩到 latent 后，用大规模 Transformer 在时空 token 上建模。

但 Transformer 也不是没有代价。Self-Attention 的计算复杂度通常随 token 数量平方增长：

$$
\mathcal{O}(N^2)
$$

视频 token 数量远大于图像，因为多了时间维度，所以长视频和高分辨率视频会带来显存和计算压力。工程上常通过 patch 化、window attention、factorized attention、稀疏 attention、latent 压缩等方式降低成本。

面试中可以总结为：**DiT 是用 Transformer 做扩散噪声预测主干的架构。视频生成转向 Transformer，是因为 Transformer 更适合大规模时空 token 建模、长程依赖和多模态条件融合；代价是 token 数量大时计算成本更高。**

<h2 id="面试问题u-net-和-dit-的核心差异是什么">面试问题：U-Net 和 DiT 的核心差异是什么？</h2>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐⭐⭐ (5/5)**

U-Net 和 DiT 的核心差异在于：**U-Net 是卷积式、多尺度、编码 - 解码结构；DiT 是 token 化、Transformer 式、全局注意力结构。**

可以从几个维度对比：

| 对比维度 | U-Net | DiT |
|----------|-------|-----|
| 基本单元 | 卷积、残差块、下采样/上采样 | Transformer block、Self-Attention、MLP |
| 数据表示 | 特征图 feature map | patch token / latent token |
| 建模方式 | 局部卷积 + 多尺度融合 | 全局 token 交互 |
| 条件注入 | timestep embedding、cross-attention、ControlNet 等 | AdaLN、FiLM、cross-attention、多模态 token 融合 |
| 优势 | 局部细节好、训练稳定、工程成熟 | 长程依赖强、扩展性好、适合大规模多模态建模 |
| 局限 | 全局建模和 scaling 能力有限 | token 多时计算和显存开销大 |

U-Net 更像是“图像处理专家”。它利用卷积归纳偏置处理局部纹理、边缘、形状和多尺度结构，对图像扩散非常有效。Stable Diffusion 早期版本就使用 U-Net 作为噪声预测主干。

DiT 更像是“通用 token 建模器”。它不强依赖卷积，而是把 latent patch 当作 token，通过 attention 建立全局依赖。这使它更适合高容量模型、长程一致性、多模态条件和视频时空建模。

在视频生成里，二者的差异会更加明显。U-Net 可以通过 3D 卷积或 temporal attention 扩展到视频，但通常仍带有较强的图像模型迁移痕迹；DiT 则可以更自然地把视频表示为时空 token，从一开始就做统一时空建模。

从工程选型看：

1. **资源有限、任务偏图像或短视频**：U-Net 更稳、更成熟。
2. **追求大规模生成质量和长程一致性**：DiT / Video Transformer 更有潜力。
3. **已有图像扩散模型需要快速视频化**：U-Net + temporal module 是常见路径。
4. **从头训练大规模视频基础模型**：DiT 类架构通常更符合趋势。

面试里可以回答：**U-Net 强在卷积多尺度去噪和局部细节恢复，DiT 强在 token 化全局建模和大规模扩展。视频生成从 U-Net 转向 DiT，本质上是从图像式局部建模走向统一时空 token 建模。**

<h2 id="面试问题fit-在视频生成架构中应该怎么理解">面试问题：FiT 在视频生成架构中应该怎么理解？</h2>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐ (3/5)**

FiT 在视频生成架构中可以理解为一种**更强调全图、全帧或完整 latent 表示的 Transformer 式扩散主干思想**。在不同语境中，FiT 的具体名称和实现可能略有差异，但在面试中更重要的是理解它代表的架构方向：**减少卷积式局部处理，更多依赖 Transformer 对完整视觉 token 序列进行统一建模。**

如果说 U-Net 是典型的多尺度卷积扩散主干，DiT 是标准的 patch-token Transformer 扩散主干，那么 FiT 可以从更工程化的角度理解为：**面向完整图像或完整视频帧 latent 的 Transformer 建模方式，强调全局上下文、统一 token 表示和高分辨率生成能力。**

在视频生成中，FiT 类思想通常包含几个特征：

1. **全帧 token 化**：把每一帧或视频 latent 划分为 token，不再只依赖局部卷积特征图。
2. **统一时空建模**：通过 Transformer 同时处理空间 token 和时间 token。
3. **高分辨率友好**：更关注全局结构、构图和长距离依赖，对复杂画面生成更有优势。
4. **条件融合自然**：文本、图像、音频、首帧等条件都可以作为 token 或调制信号进入主干。

一个简化理解是：

```text
Video Latent -> Patch / Frame Tokens -> Transformer Blocks -> Noise / Velocity Prediction
```

FiT 和 DiT 的关系可以这样把握：**DiT 是扩散 Transformer 的典型范式，FiT 更强调完整视觉表示或全图/全帧级 Transformer 建模。** 二者不是完全割裂的概念，而是都属于“扩散模型 Transformer 化”的方向。

需要注意的是，面试中不要把 FiT 简单说成“某个固定结构等价于 DiT”。更稳妥的说法是：FiT 体现的是 full-image / full-frame Transformer 的建模思想，它强调把完整视觉内容纳入统一 Transformer 表示中，从而提升全局建模和高分辨率生成能力。

它的挑战同样来自 token 数量。视频的 token 数量通常可以近似理解为：

$$
N \approx \frac{T}{p_t} \cdot \frac{H}{p_h} \cdot \frac{W}{p_w}
$$

其中 $p_t, p_h, p_w$ 分别是时间和空间 patch 尺度。随着帧数和分辨率增加，Transformer 的计算成本会快速上升。

面试里可以总结为：**FiT 可以理解为全图或全帧 Transformer 式扩散主干，强调完整视觉 token 序列上的统一建模。它和 DiT 同属扩散 Transformer 化趋势，优势是全局建模和高分辨率表达更强，挑战是视频 token 数量巨大、计算成本高。**

<h2 id="面试问题adaln--film-类调制为什么在-dit-中很常见">面试问题：AdaLN / FiLM 类调制为什么在 DiT 中很常见？</h2>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

AdaLN / FiLM 类调制在 DiT 中很常见，核心原因是：**Transformer 主干需要一种稳定、轻量、全局有效的方式注入扩散时间步和条件信息，而归一化调制非常适合做这件事。**

FiLM（Feature-wise Linear Modulation）的基本形式是对特征做通道级缩放和平移：

$$
\mathrm{FiLM}(h) = \gamma(c) \odot h + \beta(c)
$$

其中 $h$ 是中间特征， $c$ 是条件向量， $\gamma(c)$ 和 $\beta(c)$ 分别由条件生成。

AdaLN（Adaptive LayerNorm）则通常是在 LayerNorm 后进行条件调制：

$$
\mathrm{AdaLN}(h, c)
= \gamma(c) \odot \mathrm{LN}(h) + \beta(c)
$$

在 DiT 中，条件 $c$ 往往由扩散时间步 embedding、文本条件、类别条件或其他全局条件组合而成。模型通过一个小的 MLP 预测每个 Transformer block 所需的调制参数。

这样做有几个好处：

1. **适合 Transformer 结构**：Transformer 本身大量使用 LayerNorm，直接调制 LN 后的特征非常自然。
2. **条件注入稳定**：相比直接拼接条件 token，调制方式更轻量，不会显著增加 token 长度。
3. **全局控制能力强**：扩散时间步、类别、风格等全局条件可以影响整个 block 的计算。
4. **便于扩展多条件**：文本 embedding、timestep embedding、类别 embedding 可以融合后统一生成调制参数。
5. **训练更稳定**：许多 DiT 实现会使用 AdaLN-Zero 等初始化策略，让模型初始状态接近恒等映射，降低训练不稳定风险。

在扩散模型中，时间步 $t$ 是非常关键的条件。因为不同噪声等级下，模型要执行的去噪行为不同：早期步骤更关注全局结构，后期步骤更关注细节恢复。AdaLN / FiLM 可以让每一层根据 $t$ 动态调整特征处理方式。

对于文本或多模态条件，AdaLN / FiLM 通常和 cross-attention 互补使用：cross-attention 适合 token 级语义读取，AdaLN / FiLM 适合全局风格、时间步、类别或整体条件调制。

面试中可以回答：**AdaLN / FiLM 是 DiT 中常见的条件注入机制，它通过条件生成缩放和平移参数来调制 Transformer 特征。这样既能稳定注入 timestep 和全局条件，又不会显著增加 token 数量，是 DiT 类扩散模型中非常重要的条件控制手段。**
